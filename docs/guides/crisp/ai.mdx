
# 智能回复

在收到 Crisp 消息后，系统将首先匹配关键词回复，然后再进行智能 AI 回复。

**请注意：智能回复将消耗您的余额，费用根据套餐的不同而异。**
**综合费用大约为每条文字回复：￥ 0.002 左右，价格十分便宜。**

## AI 消息体构建如下：

1. 您配置的提示词
2. 您创建的知识库
3. 用户的 Crisp 元数据（以下信息均来自Crisp，并非我们主动获取）
   1. 用户信息（用户/您的网站上报）
   2. 地理位置
   3. 操作系统
   4. 浏览器
   5. 附加信息（您的网站上报）
4. 用户发送的消息 + 发送时间

## AI 提示词

合理使用 AI 提示词可以显著提高客户沟通的效率。以下是我们提供的提示词参考：

```
作为简体中文客服，请始终以温柔、耐心的态度回复所有消息，确保每次交互中用户感受到尊重和理解。
请仅使用纯文本回复，避免使用格式化语法。
积极倾听用户的问题，耐心询问细节，并提供清晰、详细的回答或指导。
优先参考提供的内容，以提供准确解答；若知识库无相关信息，可以指导用户前往这里查看：[网址]
在每次互动中，确保为用户提供友好、积极的支持体验。

当第一次回复用户时，需加上这些文字来提示用户可以使用关键词回复：回复“菜单”获取帮助，人工客服可能短时间内无法接入，请耐心等待回复或者使用菜单系统或查阅我们的知识库。
```

使用这些提示词可以帮助您更好地与客户沟通，提升服务质量。

## AI 知识库

您可以将您的知识库转储为嵌入模型，当然您只需要输入知识库名称和内容即可，仅需消耗很低的余额即可。

## 总结

您可以只使用提示词功能，而不使用知识库功能。
但是您如需使用知识库功能，需要搭配提示词功能来让 AI 知道它是谁。


|        | **知识库嵌入**                                                                | **提示词回复**                                                           |
| ------------ | ----------------------------------------------------------------------------- | ------------------------------------------------------------------------ |
| **原理**     | 将知识库向量化，用户提问时先检索相似片段，再把检索结果拼接到提示词中交给 AI   | 把整个知识库一次性写入AI提示词，AI 每次回答都基于完整上下文              |
| **价格对比** | 只消耗 **问题 + 检索到的部分片段** 的 token，通常更省钱（知识库大时优势明显） | 需要把 **整个知识库** 都放进上下文，每次调用都很耗 token，库越大成本越高 |
| **响应速度** | 检索 + 回复，前置步骤多一点；但上下文更小 → 模型推理更快                      | 无检索步骤；但上下文极大时，模型处理速度会明显变慢                       |
| **内容质量** | 回答更 **聚焦、相关**，只基于匹配度高的内容，不易跑题                         | 可能基于 **全部提示** 生成，信息更全，但也容易出现冗余、跑题或“幻觉”     |
